👆 Two Pointers Pattern (Python)

Core Concept: Imagine you're looking for a pair of numbers in a sorted list. Instead of checking every combination (which would be slow), we use two markers that start at opposite ends and move toward each other. This reduces the time complexity from O(n²) to O(n)!
# Python implementation of Two Pointers def two_pointers(arr, target): # Start with pointers at both ends left = 0 right = len(arr) - 1 while left < right: current_sum = arr[left] + arr[right] # Found the perfect pair! if current_sum == target: return [left, right] # If sum is too small, move left pointer right elif current_sum < target: left += 1 # If sum is too big, move right pointer left else: right -= 1 # If we exit the loop, no pair found return [-1, -1] # Example usage: # print(two_pointers([1, 2, 3, 4, 6], 6)) → [0, 4]

🧠 Understanding the Logic

This pattern works because the array is sorted. When we move the left pointer right, we're asking for a larger sum. When we move the right pointer left, we're reducing the sum. This intelligent pointer movement helps us find the answer in linear time!

🪟 Sliding Window Pattern (Python)

Core Concept: Picture a window sliding over your array. Instead of recalculating everything each time, we maintain a running total by subtracting the element leaving the window and adding the new element entering it. This is perfect for finding subarrays with specific properties!
# Python implementation of Sliding Window def max_subarray_sum(arr, k): # Handle edge case if len(arr) < k: return -1 # Calculate first window sum window_sum = sum(arr[:k]) max_sum = window_sum # Slide the window through the array for i in range(k, len(arr)): # Subtract outgoing element, add incoming element window_sum += arr[i] - arr[i - k] max_sum = max(max_sum, window_sum) return max_sum # Example usage: # print(max_subarray_sum([2, 1, 5, 1, 3, 2], 3)) → 9

🚪 How the Window Slides

The key insight is that we don't need to recalculate the entire window sum each time. By maintaining a running total and adjusting it as the window moves, we save computation time. This reduces the time complexity from O(n*k) to O(n) - a huge improvement!

Time Complexity: O(n) | Space Complexity: O(1)

🐢🐰 Fast & Slow Pointers (Python)

Core Concept: Also known as "Floyd's Tortoise and Hare", this pattern uses two pointers moving at different speeds. The slow pointer moves 1 step at a time while the fast moves 2 steps. This helps detect cycles and find middle elements efficiently.
# Python implementation for Cycle Detection class Node: def __init__(self, value): self.value = value self.next = None def has_cycle(head): # Initialize both pointers at head slow = head fast = head while fast and fast.next: slow = slow.next # Tortoise moves 1 step fast = fast.next.next # Hare moves 2 steps if slow == fast: return True # Cycle detected! return False # No cycle found

⏱️ Why Different Speeds?

If there's a cycle, the fast pointer will eventually "lap" the slow pointer, causing them to meet. If there's no cycle, the fast pointer will reach the end first. This approach uses O(1) space compared to O(n) space with a hash set!
